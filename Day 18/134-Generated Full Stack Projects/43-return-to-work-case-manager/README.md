# Return-to-Work Case Manager

## Summary
Return-to-Work Case Manager is a full-stack workspace for managing return to workflows with live dashboards, seeded operational records, and room to extend into production features.

## Problem
Return-to-Work Case Manager addresses a real-world gap in recovery case management, where teams still rely on spreadsheets, email, or delayed follow-up.

## Frontend Features
- ReturntoWork Case Manager case timeline
- ReturntoWork Case Manager restriction cards
- ReturntoWork Case Manager task board

## Backend Features
- ReturntoWork Case Manager case APIs
- ReturntoWork Case Manager milestone workflows
- ReturntoWork Case Manager note endpoints

## Tech Stack
- HTML
- CSS
- JavaScript
- Node.js
- Express
- PostgreSQL-ready schema

## Difficulty
Medium-Hard

## Run Locally
1. In `backend`, run `npm install` and then `npm run dev`.
2. In `frontend`, run `npm install` and then `npm run dev`.
3. Open the frontend URL shown by `serve`.

## API Endpoints
- `GET /api/health`
- `GET /api/overview`
- `GET /api/work-items`
- `GET /api/metrics`
- `POST /api/work-items`
